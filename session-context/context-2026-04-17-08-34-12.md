## Part 4 — ADR-0021 process-consent-event pipeline (commit `23a53a7`)

### ADR draft

"enforced by UNIQUE + ON CONFLICT DO NOTHING". Test 10.3 (Vault break)

### Execution — Edge Function

`supabase/functions/process-consent-event/index.ts` (~262 lines):
1. POST body `{ consent_event_id }`.
2. Fetch consent_events row; fast-path skip if artefact_ids non-empty.
3. Fetch consent_banners.purposes JSONB.
4. For each purpose in purposes_accepted with matching
   `purpose_definition_id`: fetch purpose_definition, compute
   `expires_at = now() + default_expiry_days`, INSERT consent_artefact.
5. On `23505` unique violation: fetch existing artefact_id, push to
   local list (sibling invocation won the race).
6. INSERT consent_artefact_index per newly-created artefact.
7. UPDATE consent_events.artefact_ids with guarded `eq('artefact_ids', '{}')`.
