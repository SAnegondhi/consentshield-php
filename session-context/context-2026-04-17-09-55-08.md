## Key architectural decisions / amendments this session

1. **Admin SELECT policies on public tables** (migration 20). `admins_select_all` gated on `admin.is_admin()` adds admin-JWT read access to 15 operational tables without widening customer isolation (customer JWTs don't carry the claim).
2. **Single admin-config KV key** (not 3 as in schema doc). `admin:config:v1` holds the full snapshot — kill_switches + active tracker signatures + published sectoral templates + suspended_org_ids. Saves 2 KV reads per banner request.
3. **`noopBannerResponse()` serves both kill-switch and per-org suspension**. Customer's `<script src="...">` continues to execute a valid-JS file.
4. **`cron_secret` → `cs_orchestrator_key`** (Sprint 3.2 fix migration). All admin/customer/DEPA crons now use a single vault secret.
5. **`create_support_ticket` is the single customer-facing admin RPC.** SECURITY DEFINER but deliberately skips `admin.require_admin` — called from the customer app's public support ticket endpoint. Uses oldest admin row as nominal audit author; pre-bootstrap no audit row is written (acknowledged deviation).
6. **`admin.refresh_platform_metrics` guarded with `to_regclass`** for pre-DEPA environments. Terminal B's DEPA tables existed before refresh was ever called in dev, so the guard was defensive only; it stays.
7. **Cookie-backed impersonation session** (not a session swap). Admin stays on admin URL; audit log captures start + end via the existing RPC audit-row path; intermediate-action linkage deferred.
