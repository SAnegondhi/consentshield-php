# ConsentShield Java SDK

Official Java client + Spring Boot starter for the [ConsentShield Compliance API](https://api.consentshield.in/v1).

- **Maven Central**: `com.consentshield:consentshield-java-spring-boot-starter:1.0.0` (Spring Boot starter)
- **Maven Central**: `com.consentshield:consentshield-java:1.0.0` (raw client; non-Spring callers)
- **Source**: <https://github.com/SAnegondhi/consentshield-java>

## Install (Spring Boot)

```xml
<dependency>
  <groupId>com.consentshield</groupId>
  <artifactId>consentshield-java-spring-boot-starter</artifactId>
  <version>1.0.0</version>
</dependency>
```

```yaml
# application.yml
consentshield:
  api-key: ${CONSENTSHIELD_API_KEY}
  fail-open: false
```

```java
@RestController
public class Marketing {
    private final ConsentShieldClient client;
    public Marketing(ConsentShieldClient client) { this.client = client; }

    @PostMapping("/send")
    public ResponseEntity<?> send(...) {
        // verify, gate, return
    }
}
```

## Install (non-Spring)

```xml
<dependency>
  <groupId>com.consentshield</groupId>
  <artifactId>consentshield-java</artifactId>
  <version>1.0.0</version>
</dependency>
```

```java
ConsentShieldClient client = ConsentShieldClient.builder()
    .apiKey(System.getenv("CONSENTSHIELD_API_KEY"))
    .build();
```

## Compliance contract

| Outcome | Default behaviour |
|---|---|
| 4xx response | `ConsentShieldApiException` — ALWAYS surfaces, never retried |
| Per-attempt timeout | `ConsentShieldTimeoutException` — NEVER retried |
| 5xx + transport failure (fail-CLOSED, default) | `ConsentVerifyException` with `FailureCause` discriminator |
| 5xx + transport failure (fail-OPEN) | `ConsentVerifyOutcome.isOpen() == true` with `getOpenCause()` set |

Retry budget: 3 retries on 5xx + transport with 100 / 400 / 1600 ms exponential backoff. Per-attempt timeout 2 s. Configurable via `consentshield.timeout` and `consentshield.max-retries`.

`X-CS-Trace-Id` is round-tripped on every call; `exception.getTraceId()` returns the server-side trace id when available.

## Source layout

- `generated/` — generated by `openapi-generator-cli@v7.10.0` from `app/public/openapi.yaml` in the upstream monorepo. Do NOT hand-edit.
- `wrapper/` — Spring Boot auto-config + retry interceptor + exception hierarchy.
- `examples/spring-boot-marketing-gate/` — runnable Spring Boot 3 example.
