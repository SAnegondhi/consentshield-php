Stateless Oracle: Buffer → Deliver → Delete
The core data architecture principle. ConsentShield processes compliance data and delivers it to customer-owned storage. It never holds the canonical record. The buffer exists only to guarantee delivery — once the customer's storage confirms the write, ConsentShield's copy is hard-deleted.
ConsentShield's database is an operational state store, not a compliance record store. The canonical record always lives in the customer's storage.
Deletion Orchestration Flow
When a data deletion is triggered — by an erasure request, expired retention period, or consent withdrawal — ConsentShield issues deletion commands to all connected third-party systems and collects confirmation receipts. The receipts are the evidence that data was actually removed, not just that removal was requested.
Consent Withdrawal Verification Loop
When a user withdraws consent, ConsentShield doesn't just fire a webhook and move on. It schedules three automated scans to verify that the relevant trackers actually stopped. Client-side monitoring (banner script) catches dynamic trackers. Server-side scans (HTTP checks) catch hardcoded scripts.
Tracker Detection Engine
The banner script v2 observes third-party scripts loading on the customer's website, classifies them against a curated signature database, and compares classifications against the user's consent state. Mismatches are violations.
Database Schema — Entity Relationships
All tables with their relationships and data classification. Blue tables hold operational state (permanent). Orange tables hold user data (buffered, delivered to customer storage, then deleted). The organisations table is the root of all multi-tenant data — every row in every table carries an org_id enforced by Row-Level Security.
Operational state — lives in ConsentShield permanently
User data buffer — delivered to customer storage, then hard-deleted
All tables carry org_id with Row-Level Security. The RLS policy is: every query filters by the org_id in the user's JWT. There is no way to forget it — it runs at the database level, not in application code.
