The DEPA Difference

One artefact per purpose.
Independently revocable.

The other tools treat consent as one row with a list of accepted purposes. When a user withdraws one of them, the row is mutated — evidence destroyed — or a cryptic "withdrawn" event is written with no data scope to act on.

ConsentShield creates one independently revocable, time-bounded, data-scoped artefact per purpose. When a user withdraws insurance marketing consent, the system stops exactly that flow — the bureau reporting, the co-lending share, the analytics, everything else continues. The deletion cascade is surgical, receipted, and exportable.

DEPA · Data Empowerment & Protection Architecture

Why ConsentShield
isn't another GDPR tool
in a sari.

DEPA is India's own consent architecture — designed by iSPIRT, implemented by ABDM, and the model the DPDP Act is converging toward. It has five structural requirements. Every India-focused competitor uses a GDPR-adapted event model that satisfies none of them.

See the product → Why banks need this
The problem in one paragraph
A private bank customer opens an account. At account opening, the bank obtains consent for: bureau reporting, insurance cross-sell, a co-lending fintech share, WhatsApp marketing, and analytics. Two years later, the customer decides they no longer want insurance cross-selling. Under DPDP Section 6, they have the right to withdraw that specific consent without affecting any other purpose. The bank must stop exactly that one data flow — leaving bureau reporting, co-lending, WhatsApp, and analytics intact — and prove it did. → No GDPR-adapted consent tool can execute this. ConsentShield's DEPA-native artefact model is the only architecture that can.
The two architectures, side by side

One mutates evidence.
The other is evidence.

Competitor default

GDPR event model

One row per consent decision with an array of accepted purposes. When a user withdraws one purpose, you have two choices — both bad.

// Before withdrawal { user_id: "user_xyz", purposes_accepted: [ "bureau_reporting", "insurance_marketing", "whatsapp_marketing", "analytics" ] }

Option A: mutate the array in place. Audit trail destroyed.
Option B: write a withdrawal event with no data scope. Deletion cascade flies blind.

ConsentShield

DEPA artefact model

One artefact per purpose, each with its own data scope, expiry, revocation path, and connector map. The withdrawal is surgical.

// Five artefacts from one interaction cs_art_01HXX2 { purpose: "insurance_marketing", data_scope: ["email", "name", "dob", "account_type"], authorises: "bancassurance_conn", expires_at: "2027-01-15", status: "active" }

Revocation writes an immutable artefact_revocations row. The deletion cascade reads the exact fields and exact connector. Bureau continues. Analytics continues. Audit trail intact.

The five principles

DEPA's structural requirements,
enforced at the schema.

Not features added to a GDPR tool. The foundational data model — purpose-scoped, time-bounded, independently revocable artefacts — is how ConsentShield was built from day one.

01

Consent is purpose-scoped and machine-readable

A consent artefact is not a checkbox event. It is a structured object binding a data principal, data fiduciary, purpose, data scope, and time window — each independently referenceable.

→ Enforced via the consent_artefacts table
02

Each purpose's consent is independently revocable

Revocation of one purpose must not affect others. The revocation is a discrete, timestamped, immutable event on the specific artefact — not an update to the original record.

→ Enforced via artefact_revocations
03

Consent has defined scope — which data fields

The artefact declares the specific data fields the consent covers. The deletion cascade knows exactly which fields in which systems are authorised by which artefact.

→ Enforced via purpose_definitions.data_scope
04

Consent has explicit expiry

Every artefact has an expires_at. After that timestamp, the data flow is no longer permitted, even without explicit withdrawal. Re-consent is required.

→ Enforced via consent_expiry_queue
05

Data flows stop when the purpose is fulfilled

The artefact has a lifecycle. When the purpose is fulfilled, or the artefact expires or is revoked, the data flow stops. This is enforced, not merely logged.

→ Enforced via purpose_connector_mappings + cascade
06

Chain of custody is queryable

Grant → revocation → deletion request → receipt. Four tables linked by one artefact ID. When the DPB asks what happened on a specific date, the answer is one query, not a forensic reconstruction.

→ ConsentShield extension · beyond baseline DEPA

Pricing & plans · common questions

Is there really no credit card needed for the trial?
+
No card. The 14-day trial is a working installation of ConsentShield with the full Growth feature set. At day 11 we'll email you a billing setup link if you want to continue. If you don't, the trial expires and your data is purged within 30 days per our retention policy.
What counts as a "web property"?
+
Each unique apex domain you install our banner on. yourcompany.com and yourcompany.io are two properties. Subdomains under one apex (app.yourcompany.com, marketing.yourcompany.com) count as one.
What's a deletion connector and how do they work?
+
A connector is a pre-built integration with a downstream system that can receive a deletion request and return a signed receipt. When an artefact is revoked, the relevant connectors are called automatically. Mailchimp and HubSpot are shipping today; CleverTap, Razorpay, WebEngage, MoEngage, Intercom, Freshdesk, Shopify, WooCommerce, and Segment land through Q3 2026; Zoho CRM, Freshworks CRM, Zendesk, Campaign Monitor, and Mixpanel follow in Q4 2026. A generic webhook covers everything else from day one.
Why is BFSI pricing higher than the core SaaS plans?
+
BFSI tiers include the Regulatory Exemption Engine (DPDP × RBI × SEBI × PMLA conflict resolution), third-party data consent flows for nominees and guarantors, dual/triple breach notification timelines, and the mobile consent SDK. These capabilities are not in the core product because most SaaS customers don't need them — but for a digital NBFC, they're table stakes.
Can I switch from Starter to Growth mid-month?
+
Yes. Upgrades are immediate and prorated. Downgrades take effect at the next billing cycle.
What's the data residency story?
+
All operational data is stored in Supabase Mumbai (ap-south-1). Edge-rendered banner content is served from Cloudflare's Mumbai, Chennai, and Bengaluru POPs. For Zero-Storage deployments (BFSI Growth, Banking, Healthcare), data lives entirely in your own infrastructure — we hold only a validity index.
How do you handle GST?
+
All prices are exclusive of 18% GST. Invoices are generated automatically with your GSTIN and are CGST/SGST or IGST as appropriate based on your state of registration.
