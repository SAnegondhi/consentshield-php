Test fixtures for ConsentShield

Demo Sites

Five fake customer sites, each with its own banner configuration and tracker mix. Use these to exercise banner rendering, consent event ingestion, tracker-observation monitoring, and the enforcement dashboard on the ConsentShield admin app.

/ecommerce

A mock online store. Full purpose set (analytics + marketing + personalisation). After consent, loads GA4, Meta Pixel, and Razorpay (functional, never a violation).

bottom-bar · monitoring on

/saas

A B2B dashboard. Analytics + personalisation only — no marketing cookies. Loads Segment, Mixpanel, Intercom (functional). Banner is a modal, not a bar.

modal · monitoring on

/blog

Smallest possible configuration. Analytics only (GA4). Corner banner. Use this to test minimal-banner rendering and single-tracker detection.

bottom-right · monitoring on

/healthtech

No optional cookies at all. Banner renders once per version, monitoring is off. Use this to verify the "no tracking" path does not accidentally fire observations.

bottom-bar · monitoring off

/violator

Looks like a normal site by default. Append ?violate=1 and it loads GA4 and Meta Pixel before consent — should surface as violations in the enforcement dashboard.

bottom-bar · monitoring on · ?violate=1
